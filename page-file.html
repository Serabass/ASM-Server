Assembly Web Server (File-based)

This application is written in x86-64 Assembly language.

HTML page is read from disk at runtime - no recompilation needed to update content!

Technologies Used:

x86-64 Assembly - Low-level system programming
NASM (Netwide Assembler) - Assembler for x86 architecture
Linux System Calls - Direct kernel interface (socket, bind, listen, accept, open, read, close)
Docker - Containerization platform
Docker Scratch - Empty base image (no OS)
Kubernetes - Container orchestration (deployment ready)

Size Information:

Docker Image: 11.8kB
Binary: 8.5KB (8704 bytes)
HTML File: 3KB (3083 bytes, separate from binary)
Note: Binary is smaller because HTML is not embedded. HTML file can be updated without recompiling!

No frameworks. No libraries. Just pure assembly and syscalls.
